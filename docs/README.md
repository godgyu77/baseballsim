# 📚 문서 디렉토리

이 디렉토리는 프로젝트의 모든 문서를 카테고리별로 정리한 곳입니다.

## 📂 디렉토리 구조

```
docs/
├── patches/          # 패치 노트 및 업데이트 내용
├── reports/          # 최적화, 성능, 기능 상태 보고서
├── analysis/         # 진단 리포트 및 버전 비교 분석
├── strategies/       # 구현 전략 및 설계 문서
└── structure/        # 프로젝트 구조 문서
```

## 📋 카테고리 설명

### 📝 patches/
패치 노트 및 버전별 업데이트 내용을 보관합니다.
- 파일명 형식: `PATCH_NOTES_v{버전}_{날짜}.md`

### 📊 reports/
프로젝트의 최적화, 성능, 기능 상태 등에 대한 보고서를 보관합니다.
- 파일명 형식: `{제목}_REPORT_{날짜}.md`

### 🔍 analysis/
프로젝트 구조 분석, 버전 비교 등 진단 리포트를 보관합니다.
- 파일명 형식: `{제목}_ANALYSIS_{날짜}.md`

### 🎯 strategies/
구현 전략, 설계 문서, 기술 문서를 보관합니다.
- 파일명 형식: `{제목}_STRATEGY_{날짜}.md` 또는 `{제목}_IMPLEMENTATION_{날짜}.md`

### 🏗️ structure/
프로젝트 구조 및 아키텍처 문서를 보관합니다.
- 파일명 형식: `{제목}_STRUCTURE_{날짜}.md`

## 📅 파일명 규칙

모든 문서 파일은 다음 형식을 따릅니다:
- `{카테고리}_{제목}_{버전 또는 날짜}.md`

예시:
- `PATCH_NOTES_v2.0.0_2025-12-01.md`
- `API_OPTIMIZATION_REPORT_2025-12-01.md`
- `PROJECT_STRUCTURE_ANALYSIS_2025-12-01.md`

## 🔄 업데이트 가이드

새로운 문서를 추가할 때:
1. 적절한 카테고리 폴더를 선택하세요
2. 파일명에 날짜(YYYY-MM-DD)를 포함하세요
3. 버전 정보가 있으면 버전 번호도 포함하세요
4. 이 README.md를 업데이트하세요 (선택사항)
